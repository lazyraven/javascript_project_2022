Interview Cheat Sheet — React · Vue · Node · DevOps · AWS · CI/CD

Flashcard-style Q + short answers — organized basic → intermediate → advanced
Flashcards

React.js

QuestionShort Answer
What is React?A JavaScript library for building user interfaces using components.
Difference between React and other frameworks?React focuses only on UI; Angular/Vue are full frameworks.
What are components?Independent UI pieces, functional or class-based.
What are props and state?Props → external input (read-only). State → internal data that changes over time.
What is JSX?Syntax to write HTML-like code within JavaScript.
What are hooks?Functions (e.g., useState, useEffect) that add state and lifecycle to functional components.
What is Virtual DOM?In-memory DOM representation that updates UI efficiently by diffing.
Controlled vs Uncontrolled components?Controlled: React manages the form state. Uncontrolled: the DOM manages state, accessed via refs.
What is React.memo?HOC to memoize a component and avoid re-renders when props are unchanged.
Difference between useCallback and useMemo?useCallback memoizes functions; useMemo memoizes computed values.
What is Context API?API for sharing state globally across component tree without prop drilling.
Redux vs Context API?Redux: centralized state management with middleware. Context: simpler, built-in, suited for small-scale state.
What is SSR (Server-Side Rendering)?Rendering UI on the server (e.g., Next.js) for faster first paint and SEO benefits.

Vue.js

QuestionShort Answer
What is Vue.js?A progressive framework for building user interfaces.
Vue vs React?Vue uses templates and offers two-way binding; React uses JSX and one-way data flow.
What are directives?Template markers like v-if, v-for, v-bind that add behavior to DOM elements.
What is a Vue instance?Root object that manages data, methods, and lifecycle of a Vue app or component.
What are lifecycle hooks?Methods fired at stages like created, mounted, updated, destroyed.
Computed vs Watchers?Computed: cached calculations. Watchers: run code in response to data changes.
What is Vue CLI?Tool to scaffold, develop, and build Vue projects.
What is Vuex?Centralized state management library for Vue apps (similar to Redux).
Vue 2 vs Vue 3?Vue 3 introduces Composition API, improved reactivity, and performance improvements.
What is reactivity system?Mechanism that tracks dependencies and updates only affected parts of the UI.

Node.js

QuestionShort Answer
What is Node.js?JavaScript runtime built on Chrome's V8 engine for server-side code.
Node vs Browser JS?Node runs server-side with access to OS APIs (fs, net), browser JS runs in the browser sandbox.
What is npm?Package manager for JavaScript and Node.js packages.
Event-driven architecture?Design where events and callbacks drive flow, improving concurrency without threads.
Non-blocking I/O?I/O operations run asynchronously so the main thread isn't blocked waiting for completion.
What is Express.js?Lightweight web framework for building APIs and web servers in Node.
What is middleware?Functions that process requests/responses in an Express app pipeline.
What are streams?Interfaces for reading/writing data in chunks (efficient for large data).
What is cluster module?Module to spawn worker processes to utilize multiple CPU cores.
CommonJS vs ES Modules?CommonJS uses require/exports; ES Modules use import/export and are the modern standard.
How does event loop work?Processes async callbacks in phases: timers, pending callbacks, idle, poll, check, close, etc.
What is JWT authentication?JSON Web Tokens used for stateless authentication: header.payload.signature.

DevOps

QuestionShort Answer
What is DevOps?Culture and practices that combine development and operations for faster, reliable delivery.
DevOps vs Agile?Agile: software development process. DevOps: extends Agile with ops, automation, and delivery.
What is CI/CD?CI: automated tests on merges. CD: automated delivery/deployment to environments.
What is Git?Distributed version control system for tracking code changes.
What is Docker?Container runtime to package apps with their dependencies.
What is Kubernetes?Container orchestration platform for deploying, scaling, and managing containers.
What are monitoring tools?Prometheus (metrics), Grafana (visualization), ELK stack for logs.
What is IaC?Infrastructure as Code — provisioning infra using code/tools like Terraform or Ansible.
Blue-Green vs Rolling deployment?Blue-Green: switch between two environments. Rolling: update instances gradually.
What is zero-downtime deployment?Deploy without interrupting the service—via load balancers, routing, or replication.

AWS

QuestionShort Answer
What is AWS?Amazon Web Services — cloud platform offering compute, storage, databases and more.
What is EC2?Elastic Compute Cloud — virtual machines in AWS.
What is S3?Simple Storage Service — scalable object storage.
What is RDS?Managed relational database service (MySQL, Postgres, etc.).
What is IAM?Identity and Access Management for users, roles, and permissions.
Regions vs Availability Zones?Region: geographic area. AZ: isolated datacenters within a region.
What is VPC?Virtual Private Cloud — isolated virtual network in AWS.
What is ELB/ALB?Elastic Load Balancer / Application Load Balancer — distribute incoming traffic.
RDS vs DynamoDB?RDS: relational SQL databases. DynamoDB: fully managed NoSQL key-value/document DB.
What is Lambda?Serverless compute for running code on events without provisioning servers.
What is Auto-scaling?Automatically add/remove instances based on demand or metrics.
What is CloudFormation?AWS service to model and provision resources using templates (IaC).

CI/CD

QuestionShort Answer
What is CI?Continuous Integration — frequently merging code and running automated tests.
What is CD?Continuous Delivery/Deployment — automating release of changes to environments or production.
CI vs CD?CI focuses on building/testing; CD focuses on releasing and deploying.
Why CI/CD?Enables faster, safer, repeatable releases and reduces integration issues.
Jenkins vs GitHub Actions?Jenkins: self-hosted, highly customizable. GitHub Actions: integrated with GitHub, easier setup.
What are build pipelines?Automated sequences: build → test → package → deploy.
How to add tests in CI/CD?Run unit/integration tests as pipeline steps; fail pipeline on test failures.
What is Canary deployment?Release to a small subset of users first, then progressively increase exposure.
How to manage secrets in CI/CD?Use secret stores/vaults or encrypted pipeline variables; avoid plaintext secrets.
Artifact management?Store build artifacts (binaries, images) in repositories like Artifactory or Nexus.
