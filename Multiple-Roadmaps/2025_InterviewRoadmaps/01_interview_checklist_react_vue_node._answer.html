React — Basics
Core concepts
What is React?
A JavaScript library for building component-based UIs that uses a Virtual DOM for efficient updates.
Props vs State?
Props: immutable, passed from parent. State: local, mutable inside component.
Virtual DOM?
A lightweight in-memory representation of the real DOM. React diffs it and updates minimal real DOM nodes.
Hooks (useState, useEffect)?
Functions to use state and lifecycle in functional components — e.g., useState for state, useEffect for side-effects.
Controlled vs Uncontrolled components?
Controlled: React owns input state. Uncontrolled: DOM holds value; use refs to read values.
Keys in lists?
Unique identifiers that help React reconcile lists efficiently.
React — Advanced
State mgmt, optimisations, SSR
Context API?
Provides global-ish state to avoid props drilling.
Redux vs Context API?
Redux is a full state management library (store, reducers, middleware). Context is simpler and built-in.
Memoization (React.memo, useMemo, useCallback)?
Tools to avoid unnecessary re-renders: React.memo caches components; useMemo caches values; useCallback caches functions.
Lifecycle methods?
Class lifecycles: componentDidMount, componentDidUpdate, componentWillUnmount.
SSR vs CSR?
SSR (server-side rendering — Next.js) vs CSR (client-side rendering — default React).
Vue — Basics
Core concepts
What is Vue?
Progressive JS framework for building UIs with reactive data binding.
Vue 2 vs Vue 3?
Vue 3 adds Composition API and uses Proxy-based reactivity for better performance.
Directives (v-if, v-for, v-model)?
Built-in directives to control rendering, lists, and binding.
Computed vs Methods?
Computed: cached based on deps. Methods: executed every call.
Props?
Pass data from parent to child; validated in component options.
One-way vs Two-way binding?
One-way flows parent→child; two-way (v-model) syncs both ways.
Vue — Advanced
State mgmt & architecture
What is Vuex?
Centralized store for Vue apps with state, getters, mutations, actions.
Mixins?
Reusable blocks of logic to share between components (use Composition API for modern code).
Composition API vs Options API?
Composition: functions that group logic. Options: object-based (data, methods, computed).
Vue 3 reactivity?
Proxy-based, more complete reactivity and better performance than Object.defineProperty.
Node — Basics
Server-side JS
What is Node.js?
JS runtime built on V8 for server-side apps and tooling.
Sync vs Async?
Sync blocks, async is non-blocking — use callbacks/promises/async-await.
Event Loop?
Core mechanism handling async operations in phases and queues.
Callbacks vs Promises vs Async/Await?
Callbacks: nesting; Promises: chainable; Async/Await: clearer async code using promises.
npm & package.json?
npm is package manager. package.json lists dependencies, scripts and metadata.
Express middleware?
Functions that process requests before response handlers (logging, auth, validation).
Node — Advanced
Performance & architecture
Clustering?
Run multiple worker processes to use all CPU cores and improve throughput.
Streams?
Handle data in chunks (readable, writable, transform), efficient for large IO.
process.nextTick() vs setImmediate()?
nextTick runs before the next event loop phase; setImmediate runs in the check phase after I/O.
JWT Authentication?
Stateless tokens (signed) used for authentication; store minimal data, verify signature on each request.
Child Processes?
Use child_process to run OS commands or spawn other processes.
REST vs GraphQL?
REST: multiple endpoints. GraphQL: single endpoint with flexible queries and precise data fetching.
Securing Node apps?
Use Helmet, input validation, rate limiting, HTTPS, secure headers, and proper token handling.
