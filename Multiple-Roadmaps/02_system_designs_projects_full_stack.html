Full‑Stack System Design Problems — Interview Cheatsheet

10 common interview systems with front/back/DB and key concepts. Ready to print or copy into notes.

1. Scalable URL Shortener (Bitly)

Frontend: Simple form to shorten & redirect.
Backend: Node.js API for shorten + redirect.
Database: MongoDB { shortId → longURL }.
Concepts: Hashing, DB indexing, caching (Redis).

2. Real‑Time Chat (WhatsApp / WebSocket)

Frontend: React chat UI (real‑time updates).
Backend: Node.js + WebSockets / Socket.io.
Database: MongoDB (messages collection).
Concepts: Pub/Sub, scaling WS servers, sharding messages.

3. News Feed / Social Timeline

Frontend: Infinite scroll feed.
Backend: Node.js APIs for posting & retrieving feeds.
Database: MongoDB (posts, users, followers).
Concepts: Fan‑out on write vs read, caching hot feeds, pagination.

4. E‑Commerce (Mini Amazon)

Frontend: Product listing + cart + checkout.
Backend: REST APIs (Node.js).
Database: MongoDB (products, orders, users).
Concepts: Cart design, order consistency, inventory, payment flows.

5. Scalable Auth & Authorization

Frontend: Login / Signup UI.
Backend: Node.js with JWT / OAuth.
Database: MongoDB (users, sessions).
Concepts: Session vs JWT, refresh tokens, bcrypt password storage.

6. File Upload & Sharing (Drive Lite)

Frontend: Upload UI with progress.
Backend: Node.js + S3/GCP storage.
Database: MongoDB for metadata.
Concepts: File sharding, CDN, large uploads, signed URLs.

7. Scalable Notification Service

Frontend: Notification dropdown.
Backend: Queue‑based Node.js workers.
Database: MongoDB (notifications collection).
Concepts: Kafka/RabbitMQ, retries, rate limiting.

8. Collaborative Document Editing

Frontend: Rich text editor (React + Quill).
Backend: Node.js with WebSockets.
Database: MongoDB (doc states & versions).
Concepts: Conflict resolution, OT/CRDT, versioning.

9. Ride‑Hailing (Uber/Ola Simplified)

Frontend: Map UI (Google Maps).
Backend: Node.js booking + tracking APIs.
Database: MongoDB (drivers, riders, trips).
Concepts: Matching algorithms, geo‑indexing, real‑time location.

10. Video Streaming Platform (YouTube Lite)

Frontend: Video player UI.
Backend: Node.js APIs for upload & playback.
Database: MongoDB (video metadata).
Concepts: Video chunking, CDN, adaptive streaming, recommender basics.

How to prepare (quick checklist)

Sketch architecture: Frontend, API layer, DB schema.
Discuss trade‑offs: SQL vs NoSQL, caching, consistency vs availability.
Write pseudo API contracts in Node.js and explain endpoints.
Think scaling: sharding, load balancing, queues, caching.

Practice explaining one system in 30–40 mins — interviewers evaluate clarity, trade‑offs, and scaling decisions.

// Example pseudo-API (Node.js / Express)
// POST /api/shorten { url }
// GET  /r/:shortId  -> redirect

app.post('/api/shorten', async (req,res)=>{
  const {url}=req.body; // validate
  const shortId = generateShortId(url);
  await db.collection('urls').updateOne({shortId},{ $set: {longUrl:url, createdAt: new Date()}}, {upsert:true});
  return res.json({short: `https://short.ly/${shortId}`});
});

app.get('/r/:shortId', async (req,res)=>{
  const doc = await db.collection('urls').findOne({shortId:req.params.shortId});
  if(!doc) return res.status(404).send('not found');
  res.redirect(doc.longUrl);
});
