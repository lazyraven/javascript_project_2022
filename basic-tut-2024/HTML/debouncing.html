Debouncing

Debouncing is a strategy used to improve the performance of a feature by controlling the time at which a function should be executesd

Simple words:

It delays the execution of your code until the user stops performing a certain action for a specified amount of time. It is a practice used to improve browser performance

Use cases of Debouncing:

Debouncing is commonly used in scenarios where you want to limit the rate at which a function is executed in response to user input or events. This can help prevent performance issues and improve user experience.

For example, if you have a search input field that triggers an API call on every keystroke, debouncing can be used to wait until the user has stopped typing for a certain period before making the API call. This reduces the number of requests sent to the server and improves performance.

Here are some common use cases for debouncing:

  • Search input fields: Wait until typing stops before sending API requests to prevent unnecessary calls.
  • Window resize: Trigger resize-related logic only once after resizing stops.
  • Scroll events: Respond to scrolling only after the user stops scrolling for better performance.
  • Real-time form validation or live feedback: Avoid validating on every keystroke.

⚡ Why Use Debounce?:

  • Reduces unnecessary function calls and conserves resources.
  • WImproves performance and responsiveness.
  • Prevents redundant network/API calls.

⚡Quick Summary

  • Define a debounce wrapper that clears and resets a timer.
  • Use closures to retain the timer variable across calls.
  • Pass arguments via ...args and func.apply(this, args).
  • Choose delay based on your UX needs (commonly 200–500 ms).
  • Consider immediate execution when needed.

Example:

In this example, we will create a simple input field that logs the input value to the console after a delay of 500 milliseconds when the user stops typing.

                let timeoutId;
                const inputField = document.getElementById('inputField');

                inputField.addEventListener('input', function() {
                    clearTimeout(timeoutId);
                    timeoutId = setTimeout(() => {
                        console.log(inputField.value);
                    }, 500);
                });
            
In this example, we use a timeout to delay the execution of the console log function.
                   When the user types in the input field, we clear any existing timeout using clearTimeout.
                    We then set a new timeout that will log the input value after 500 milliseconds.

⚡ Debounce vs Throttle:

Debounce and throttle are both techniques used to control the rate at which a function is executed in response to events, but they serve different purposes:

  • Debounce: Delays the execution of a function until a specified time has passed since the last event. It ensures that the function is called only once after the user stops triggering the event for a certain period.
  • Throttle: Limits the execution of a function to a fixed interval, ensuring that it is called at most once every specified time period, regardless of how many times the event occurs.
